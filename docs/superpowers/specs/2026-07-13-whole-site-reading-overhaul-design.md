# AI Chronicle 整站阅读体验重构设计

## 目标

在不下线、不降级现有事件的前提下，把 AI Chronicle 从“节点很多的资料索引”重构为“可以连续阅读、能够理解因果与路线的行业编年史”。本轮同时处理文章质量、详情页进入顺序、时间轴叙事、专题发现、搜索与编辑质量门槛，并保持现有视觉语言、双语结构和静态站架构。

## 内容改造

- 优先重写所有短、薄、模板化的事件正文，尤其是 B 级事件和 2024–2026 年新增条目。
- 每篇文章必须有事件独有的叙事中心、至少两类可核对材料、一个真实矛盾或边界，以及唯一且自然的结尾。
- 删除“史位”“编年应记下”“正确姿势”“写进对照表”等编辑元语言，减少“不是 A，而是 B”式统一判断腔。
- 清理 GPT-5.6、Devin、GitHub Copilot、Claude Code、Cursor 等文章的重复段落和多重结尾。
- 中文与英文分别重写，保留同一事实边界，但不逐句镜像。

## 页面与信息架构

- 事件页首屏只保留标题、日期、级别、组织、状态和简短史实；完整档案改为可展开区域，让主视觉与正文更早出现。
- 移动端时间轴恢复一到两行摘要；搜索入口保持可见，导航移除重复外链。
- 时间轴增加“看最新 / 从头读”双模式，并用时代章节帮助读者理解历史阶段。
- 事件页显示已有的概念、论文和可点击关系；相关事件区分前因、后续与同路线阅读。
- 专题从自动标签列表升级为少量有阶段、有必读节点的编辑路线；普通标签继续用于筛选。
- 全站搜索按事件、概念、公司、模型和专题分组，并提升精确标题命中的优先级。

## 编辑质量门槛

- 扩展编辑检查：短文清单、固定三段形状、单篇段落相似、连续正文粘连、高频编辑元语言和重复结尾。
- 自动检查只负责暴露风险；最终按同一年连续阅读至少五篇进行人工验收。
- 保留现有事实来源、站内链接、图片权利和双语完整性检查。

## 验收

- 所有现有文章继续在线，不通过降级掩盖质量问题。
- 低质量短文完成结构性重写，不再只是元数据摘要的修辞复述。
- 移动端正文在首个视口附近出现，时间轴条目具备点击理由。
- 首页、专题、搜索和事件关系形成可连续探索的阅读路径。
- `pnpm build`、编辑检查与站内链接检查全部通过。
- 提交并推送 `main`，部署 Cloudflare Pages，随后在真实 Chrome 中验证桌面和移动端。
