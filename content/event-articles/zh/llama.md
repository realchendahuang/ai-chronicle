对托管 API 来说，模型是一项远方的服务。请求越过网络，答案返回，使用者看不见权重，也不能把它拆开、缩小、带回自己的机器。LLaMA 改变的首先不是某张排行榜，而是这种距离。

Meta 在 2023 年 2 月公布了 7B、13B、33B 和 65B 四种规模的 LLaMA。论文强调一个当时很重要、后来愈发常见的判断：推理预算固定时，较小的模型看过更多 token，可能比一味增加参数更合算。结果是，一组参数量相对克制的模型在多项基准上取得了有竞争力的表现。对研究者而言，权重可以申请获得同样关键。模型从一个远端端点变成一批能够下载、检查和修改的文件。

随后发生的事并不按正式发布计划展开。权重外泄，迅速流入更广的社区。有人研究怎样降低精度而尽量保住能力，有人制作微调方法，有人让模型在消费级硬件上运行。量化、适配器、推理工具和一批衍生模型彼此推动，原本只准备面向研究者的发布变成了开放模型生态的一次加速。许多工作没有宏大的名称，只是在解决显存不够、速度太慢、安装太难这些具体问题；正是这些问题被逐个解决以后，本地运行大语言模型才从少数人的实验变成一条真正存在的路线。

这里的“开放”始终带着需要辨认的边界。初代 LLaMA 不是按开源软件许可证发布，研究许可也不等于任何人都能自由商用。后来 Llama 2 扩大了商业使用范围，依然保留额外限制。权重可得、代码公开、训练数据透明和许可证符合开源定义，本来就是四件不同的事。LLaMA 的影响越大，人们越不能再用一个模糊的“开源”把它们混在一起。

封闭 API 提供的是秩序：版本、算力和服务由一家公司维护。权重进入社区以后，获得的是另一种速度，也接受另一种混乱。模型会被压缩、微调、误用、改名，能力和风险都不再由原发布者独自决定。LLaMA 没有终结托管模型的优势，却证明了另一条技术路线可以繁荣：模型可以留在企业的数据边界里，可以在研究者手中被拆解，也可以在没有云端许可的机器上继续变化。

从那以后，“能不能拿到权重”成为模型发布时无法绕开的问句。这句问话背后不是浪漫的自由宣言，而是一种很实际的欲望：人们不满足于隔着接口使用智能，他们还想知道它占多少显存，能不能改，能不能在自己的电脑上跑起来。
