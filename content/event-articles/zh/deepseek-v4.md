DeepSeek-V4 的发布说明一半像模型介绍，一半像迁移通知。它宣布 Pro 与 Flash 预览版上线、开放权重，也告诉开发者：`base_url` 可以不动，把模型名改成 `deepseek-v4-pro` 或 `deepseek-v4-flash`；旧的 `deepseek-chat` 与 `deepseek-reasoner` 将在 7 月 24 日退出。

换名不是表面工作。V4 把此前分开的聊天与推理入口收进同一家族，两款模型都提供思考与非思考模式。Pro 有 1.6 万亿总参数，每次激活 490 亿，面向较难的推理与 Agent 编程；Flash 有 2840 亿总参数，激活 130 亿，官方把它放在更快、更经济的位置。两者继续采用稀疏计算的思路，让保存的参数规模与一次实际动用的参数规模分离。

发布页把“一百万 token 成为官方服务默认上下文”放在最显眼的位置。这个数字使大型代码仓库、长文档与持续 Agent 记录更容易装进一次任务，也同时把问题从“能不能输入”推向“输入以后能不能始终找对、想对、做对”。DeepSeek 在页面上强调长上下文的计算与内存效率，却没有因此证明每一种百万 token 工作流都具有同样可靠性。上下文窗口是容器，不是理解质量的保证书。

Agent 编程也是这次发布的中心措辞。DeepSeek 称 V4-Pro 在 Agent Coding 基准上达到开放模型的领先水平，Flash 在简单 Agent 任务上接近 Pro，并列出与 Claude Code、OpenClaw、OpenCode 的集成。这些属于发布方的评测与兼容性说明；“媲美顶级闭源模型”同样是官方判断。预览版真正交到开发者手里的，是权重、技术报告、API 与可运行的模型名，长期执行的稳定性仍需在具体工具、权限和任务中检验。

V4 因而延续了 R1 之后那条开放路线，却把竞争场地从一道可验证的数学题推到更杂乱的工程环境。模型要记住更长的历史，选择是否思考，调用外部工具，还要在费用允许的范围内持续运行。Pro 与 Flash 的分工也承认：不是每个任务都值得动用同样多的激活参数。

发布说明最后留下了一句很日常的话：保留地址，只换模型。前沿能力经常以巨大的参数数目被记住，对真正维护系统的人，它抵达时却可能只是配置文件里一行字符串改变，随后是旧别名倒计时。模型史与运维史在这里落到同一页上。
