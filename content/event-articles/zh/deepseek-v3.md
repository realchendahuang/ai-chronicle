DeepSeek-V3 技术报告里最广为流传的内容，不是一张基准图，而是一张四列成本表。

预训练 266.4 万 H800 GPU 小时，上下文扩展 11.9 万，后训练 5000；合计 278.8 万 GPU 小时。报告按每 GPU 小时 2 美元估算，得到 557.6 万美元。表格把一个前沿大模型压缩成几行可以相加的数字，在长期依赖外部猜测的行业里，这种明确本身就具有冲击力。

但表格下面的脚注与总数同样重要。它计算的是 DeepSeek-V3 正式训练的 GPU 时间，不包括此前为架构、算法和数据所做的研究与消融实验，也不等于人员、数据管线、集群建设、失败尝试和推理服务的全部成本。557.6 万美元是依据报告假设换算出的训练费用，不是从公司账簿里找出的项目总价。数字一旦离开这个边界，效率故事就会迅速变成神话。

真正值得追问的是，它为什么能把表格写成这样。DeepSeek-V3 是一个 6710 亿总参数的混合专家模型，每个 token 激活约 370 亿参数。并非所有参数都在每次计算中工作，这让模型容量与实际计算量不必同步增长。它沿用 Multi-head Latent Attention 和 DeepSeekMoE，又加入不依靠主要辅助损失的负载均衡策略，减少为了让专家平均忙碌而损伤模型能力的代价；多 token 预测目标让训练不只盯着下一个 token。FP8 混合精度、通信与计算重叠、跨节点通信内核等系统工程，则负责让 2048 张 H800 上的大规模训练尽量少把时间浪费在等待彼此。

报告称，模型在 14.8 万亿 token 上预训练，全程没有遇到不可恢复的损失尖峰，也没有回滚。随后完成上下文扩展、监督微调和强化学习。厂商公布的评测显示，它在多项代码、数学和知识基准上超过其他开放模型，并接近若干闭源模型。模型权重可供下载，使外部开发者获得了比 API 更大的部署空间；训练数据与完整生产过程并没有因此全部公开，“开放权重”仍比“完整开源”窄。

DeepSeek-V3 没有证明一张小额支票就能训练前沿模型。它把成本效率拆成架构、数值精度、路由、通信、稳定训练和硬件条件，算力仍然巨大，工程也没有变轻松。从这份报告以后，再说“模型越强，只能堆越多钱”，就必须先回答那张表格提出的问题：钱究竟花在哪一步，哪一步还有别的做法？
