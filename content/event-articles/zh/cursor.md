GitHub Copilot 已经证明行内补全有价值，开发者却仍常在编辑器、网页聊天和复制粘贴之间切换。完整代码库的理解需要额外索引与上下文选择——这些工作过去散落在人的手指上。光标停在文件里，答案却在另一个标签页生成，再被粘回来。

2024 年 8 月 22 日，Anysphere 宣布完成 6000 万美元 A 轮融资。新闻是资本事件；产品事实在更早与更后的版本里：Cursor 把模型聊天、代码库索引和生成式编辑直接放进基于 VS Code 的环境。扩展生态仍熟悉，主交互却开始变成：引用文件与符号提问、语义检索、跨多处应用修改。融资验证的是市场愿意为这条路线下注，不是某一天模型突然“会写所有代码”。

后来的 Composer 与 Agent 模式把操作范围扩到多文件与终端，时间线不应压回 8 月 22 日单点。首发叙事的核心是编辑器身份的迁移：AI Coding 从侧栏附件，变成默认工作方式的一部分。上下文收集、编辑体验与工作流速度，开始与“谁的模型更强”并列成为竞争轴。

Cursor 没有取消审查与测试。它改变的是默认路径——生成与修改发生在仓库打开的同一块屏幕上，错误也在同一块屏幕上被看见。顺手是真实的；选错上下文、误改文件同样真实。产品史应同时记下这两种真实。融资标题会过期；编辑器里多出来的那一层主交互，会留下更久。

把融资日当成产品生日并不准确，却提供了一个便于书写的锚点：此时 Cursor 已足够清晰地展示“AI 原生编辑器”路线，资本愿意为之定价。开发者关心的仍是更具体的摩擦——索引是否跟上仓库变化，生成编辑是否可撤销，Agent 模式误改时如何回退。顺手与失控只隔一步。编辑器产品的胜负，往往不在模型口号，而在这些摩擦有没有被认真对待。

VS Code 兼容降低了迁移成本，也决定了竞争必须发生在模型上下文与编辑体验上，而不是再发明一套完全陌生的 IDE。熟悉皮肤下的新主交互，是 Cursor 路线的核心修辞：你还在你的编辑器里，但工作方式已经改了。
与 Claude Code 的对照有助于理解编辑器路线的边界：Cursor 赢在降低迁移成本与上下文收集的顺手；终端 Agent 赢在贴近脚本与测试命令的真实失败。Devin 则把等待与沙箱变成第三种节奏。2024–2025 年，这三种节奏同时被市场定价，AI Coding 不再是单一功能点。

Agent Coding 专题若只写融资，会漏掉开发者每天的撤销与审查。Cursor 的史位应继续钉在主交互迁移：当聊天与多文件编辑成为默认路径，模型强弱仍重要，但索引是否跟上、误改是否可回退，同样决定去留。
