大模型服务的瓶颈，经常不像训练那样写在 loss 曲线上，而写在“下一秒还塞不塞得进一个请求”。

自回归解码每生成一个 token，就要把对应的 key 与 value 留下来，供后续步骤的注意力读取。序列越长，KV 缓存越大；并发会话一多，显存账本比参数账本更难平衡。传统做法往往为每个请求预留一段连续空间，并按可能的最大长度留余量。长度事先不知道时，余量变成空洞：碎片与过度预留并存，GPU 算力还在，新请求却已经排队或直接 OOM。ChatGPT 之后，开放权重与自建推理同时升温，这个问题从实验室脚本变成机房夜班的主诉。

2023 年 6 月 20 日，Woosuk Kwon、Zhuohan Li 等作者以博客宣布 vLLM，并把核心机制命名为 PagedAttention。灵感直接来自操作系统的虚拟内存与分页：逻辑上连续的 KV 不必物理连续；固定大小的页（block）可以映射到分散的物理显存，用页表维护对应关系。请求结束或分支复制时，可以按页回收或写时复制，而不是整段搬迁。官方材料称，在评测负载上相对 Hugging Face Transformers 等基线可获得高达约 24 倍的吞吐提升，且无需改动模型架构——数字绑定于他们的工作负载与对比设置，读作“显存利用率释放出来的并发空间”，而不是万用加速卡。

同年稍后，SOSP 论文 *Efficient Memory Management for Large Language Model Serving with PagedAttention*（arXiv:2309.06180）把设计写成系统论文：近零浪费的 KV 管理、请求内与请求间的缓存共享、以及建立在其上的高吞吐服务引擎。vLLM 以开源库形式发布，使分页 KV 从“某实验室的调度技巧”变成可以 `pip install` 的默认选项之一。开放权重模型（如 LLaMA 系）的自托管、评测平台的批量生成、以及创业公司的 API 网关，都迅速把名字写进依赖列表。

PagedAttention 没有取消注意力计算本身的代价，也不替代 FlashAttention 一类内核在算子层的优化；它解决的是另一层——缓存布局与多请求调度如何共享有限显存。服务栈因此出现清晰分工：算子内核拧单次注意力的 IO，分页管理拧批次之间的内存形状。后续推理引擎吸收相似思想，说明 2023 年夏天钉住的不是一个品牌，而是一种问题表述：LLM 服务首先是内存系统，其次才是“再堆一张卡”。

对运维者，变化具体到仪表盘。同一张 A100/H100 上可同时保持的会话数上升；前缀共享、并行采样等场景下，页级共享比复制整段缓存更自然。对研究者，评测吞吐量不再只能靠缩小 batch 或截断长度来勉强完成。边界同样清楚：错误的页大小、碎片策略与调度优先级仍会伤害尾延迟；安全隔离、多租户配额与模型本身的质量问题，分页解决不了。vLLM 的贡献是把“KV 该怎么放”从隐含约定提升为可讨论、可复现的系统层，让高并发生成从运气活变成可以对照论文与源码的工程。
