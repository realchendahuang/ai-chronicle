本地终端里的编程 Agent 占据一段连续时间。人提出任务，看它搜索与修改，遇到岔路便介入。2025 年 5 月 16 日，OpenAI 在 ChatGPT 中发布的云端 Codex 研究预览换了一种时间安排：任务被送走，各自进入隔离环境，几项工作可以同时进行，完成后再带着补丁和证据回来。

连接 GitHub 仓库后，Codex 会为每个任务准备预装代码的容器，在其中读取文件、运行命令、修改代码并执行测试。首版由 codex-1 驱动。一个 Agent 看不到另一个任务正在改什么，每项工作拥有自己的环境与过程；隔离既让并行成为可能，也把任务边界写得更清楚。修一个 bug、补一组测试、解释一段代码，可以成为三份相互独立的委托，而不是在一个对话里排队等待。开放形态是研究预览，面向具备相应 ChatGPT 权限的用户，而不是“所有开发者默认可用的通用云工程师”。

云端隔离环境使权限模型与本地终端不同：Agent 在容器里强大，对开发者笔记本的直接风险较低，但对所连接仓库与密钥注入方式提出新的治理问题。并行任务鼓励把工作切成独立工单，也惩罚切分不当——相互依赖的改动在并行中会产生合并冲突与语义冲突。研究预览阶段的产品诚实，在于强调返回补丁与日志，而不是强调“已取代工程师”。名字的复用提醒读者查日期：2021 的 API 模型与 2025 的云端任务系统，中间隔着整整一代产品哲学。

云端没有把软件工程变成一句命令。它只是把许多中间时刻从人的桌面移走。Agent 仍可能误读需求，测试仍可能覆盖不足，仓库里未写下的约定仍可能被忽略。OpenAI 因此把日志、测试结果与代码引用放在交付物里，让使用者能够追查它做过什么，而不是只接收一段完成宣言。任务在后台运行，责任没有一同被托管。并行数量也不是能力本身：十个同时失败的任务，只会更快地制造十份需要审查的噪声。

这与终端 Agent 形成了并非互斥的分工。终端适合边做边校正，尤其当目标尚在变化；云端任务适合边界较清楚、能够隔离、最后可以验收的工作。前者像共用一张工作台，后者更像提交一叠工单。模型能力可能相近，协作制度却不同：一个强调同步上下文，一个强调异步证据。它与 2021 年的 Codex API 模型同名不同物——那一代把自然语言变成可调用代码；这一代把软件任务变成可并行、可带回补丁证据的云端作业。

Codex 把队列管理带进了日常开发：哪些任务足够独立，哪些可以同时开始，什么结果算通过，冲突如何回到主分支处理。当多个 Agent 一起工作，生成代码的时间未必还是最稀缺的东西。把工作切对、把验收写清楚，以及在一批看似完成的补丁中知道哪一份值得合并，逐渐占据更重的位置。首版研究预览交付的并非“并行工程师”，而是一种更克制的产品单位：研究预览还把“证据包”设为默认交付物的一部分：不只是 diff，还有它跑过什么、测过什么、引用了哪些代码位置。这对异步委托至关重要——人不在场时，唯一能替代盯梢的就是可检查的轨迹。与 Claude Code 等终端 Agent 对照，Codex 云端版用隔离与并行换取不同的协作节奏；与 2021 年 Codex 对照，它不再满足于生成片段，而试图在仓库级任务上闭环。能力上限仍受模型与测试质量约束；产品创新首先发生在任务制度，而不是又一次补全动画。

ChatGPT 内的入口降低了尝试云端编程 Agent 的门槛，也把产品暴露给非重度终端用户——他们可能更需要清晰的验收说明。隔离环境预装仓库，使“从零 clone 与装依赖”的失败少了一截，却不能消除业务逻辑误解。并行队列的管理界面，将和 diff 审查界面一样，成为 Agent 时代的新主角。

首版研究预览交付的并非“并行工程师”，而是一种更克制的产品单位：带有独立环境、过程记录与返回包的软件任务。
