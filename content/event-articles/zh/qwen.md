2023 年 8 月 3 日，阿里云通义团队在 ModelScope 与 Hugging Face 放出 Qwen-7B 与 Qwen-7B-Chat 的代码和权重。基础模型声称以超过 2.2 万亿 token 预训练，覆盖中文、英文、代码与多语言文本；Chat 变体在指令与对齐数据上继续训练。对当时仍主要依赖封闭中文 API 或翻译腔英文模型的开发者，这是一组可以落盘、可以微调、可以塞进内网的 7B 检查点。

许可不是 Apache 式无门槛。研究使用被允许；月活用户低于一亿的商业主体可按条款直接使用，达到或超过该规模须另行申请——具体以当时 LICENSE 文本为准。准确的说法是开放权重加有条件商业条款，而不是“无限制开源”。即便如此，门槛已经低到足够让创业公司、高校与个人形成默认选择：中文分词与指令跟随不必再完全外包给云端大模型接口。企业内网部署时，数据不出域的需求与有条件商用条款可以相容，这是许多客户选择开放权重中文基座的现实理由。

技术报告（随后挂上 arXiv）把预训练数据配比、分词器、上下文长度与基准表写进可引用文献，而不仅是产品通稿。基准覆盖中英文理解、代码与数学等；发布方数字应标注为官方评测。更重要的是分发姿态：权重出现在全球模型社区常用的两个枢纽上，文档与示例按可复现实验而不是仅按云控制台演示来写。分词器对中文的切分效率直接影响同等 token 预算下的有效上下文与训练成本——这是中文基座竞争的隐线。Chat 模型的对齐数据包含中文场景下的礼貌、安全与工具使用偏好，质量参差会直接表现为产品口吻。

Qwen 的历史意义不在首个 7B 是否某张表第一，而在它把“中文开放模型”从一次新闻稿推进成可长期维护的家族承诺。此后参数档位扩展、视觉与音频变体、代码与数学专用线、许可条款调整，都属于同一产品线的连续迭代；那些章节有自己的日期，不应全部写进 2023 年 8 月 3 日。首日留下的是：中文为主的大规模预训练、可下载权重、明确（尽管带条件）的商用路径——以及一个锚点，使后续每一次权重更新都有人接住、有人微调、有人抱怨再等待下一版。

超过 2.2 万亿词元的预训练声称、70 亿参数规模、双平台分发与月活门槛许可，构成首日事实账本。账本之外的家族故事，留给各自的发布日书写。
