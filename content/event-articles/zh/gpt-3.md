过去，为一个模型准备任务，通常要在训练集和代码里动手。GPT-3 把这项工作的一部分搬进了输入框。

2020 年的论文描述了一个约 1750 亿参数的自回归 Transformer，在大规模网络文本混合语料上训练。评估被故意写成三种提示情形。零样本只有任务说明；单样本再给一个完整示例；少样本则把若干输入与答案依次写进上下文。模型读完这些文字后接着生成，不进行梯度更新。这里的“学习”发生在一次推理的上下文里：示例临时规定了答案格式、标签意义，甚至一种从未专门微调过的文字操作。

提示因此不再只是问题本身，也像一份微型数据集。论文报告 LAMBADA 少样本准确率约 86.4%；TriviaQA 从零样本约 64.3% 升到少样本约 71.2%；WebQuestions 从约 14.4% 到约 41.5%。翻译方向、文法纠正、新造词解释与简单代码补全，也被放进同一套 few-shot 协议里比较。1750 亿参数让这种现象醒目，却没有让表现整齐：某些两位数算术可靠，位数增加后迅速出错；词义判断 WiC 接近随机；常识与对抗性推理仍明显落后于专门系统。示例顺序与措辞会改变分数——作者把上下文中究竟形成了怎样的内部任务，称为尚未解释的问题。

训练语料的巨大范围带来另一层不确定。测试题可能以近似形式出现在网络数据中；作者为数据污染做了分析，却无法为所有基准给出绝对排除。偏差章节留下一句准确而沉重的概括：互联网训练的模型具有互联网尺度的偏差。规模放大了可调用的模式，也把刻板印象与有害表达一并放大。

OpenAI 没有发布 GPT-3 权重，而是通过 API 提供访问。提示既是操作模型的方法，也是产品接口：开发者不必拥有训练集群，甚至不必训练下游模型，就能用几行例子试出翻译器、分类器或写作工具。同一基础模型支撑大量应用的商业形态，在这里成形。

模型规模表列出从 1.25 亿到 1750 亿的多档对照，用以展示 few-shot 表现如何随参数量上升。训练使用数万亿 token 量级的混合语料（具体配比见论文），上下文窗口 2048。API 分发意味着外部研究者多半无法做权重级干预，却可以用提示协议在同一模型上繁殖大量应用。这种不对称——能力集中、接口开放——成为此后基础模型商业的基本形制之一。

GPT-3 把任务定义的位置从训练管线移到一段可改写的文本里。这段文本便宜、灵活、脆弱；它让模型看似立刻领会要求，也让错误随着换序、改写或一道稍难的算术显现。论文里忽高忽低的表格，留下的是这种新接口最初的手感，而不是一张全能成绩单。

少样本协议还改变了评测文化：同一模型可以用不同提示模板得到相差很大的分数，于是“提示工程”进入论文与产品讨论。数据污染分析、偏差测量与毒性样本被单独成章，说明作者意识到规模放大不只放大能力。闭源应用程序接口路线使外部无法做完整的权重科学，却催生了提示市场、代理封装与垂直应用层。此后关于开放权重与封闭接口的争论，很多句式在 GPT-3 的分发选择里已经写好草稿。

一千七百五十亿参数的训练本身依赖大规模集群与混合精度等系统工程，论文对此着墨不多，却在产业侧被反复脑补。更可核对的是评估协议与误差分析：算术、闭包与对抗样本上的失败，与少样本高分并置，防止读者把上下文学习写成万能。应用程序接口密钥成为新的生产资料，提示词成为新的配置文件——这是 GPT-3 留下的产品语法。

上下文学习把示例预算从标注平台挪到提示词。预算很便宜，方差却很大：换一个示范，分数能跳。论文用三档提示协议把方差暴露出来，而不是藏进单次最佳运行。一千七百五十亿参数是放大器，放大器后面的接口形状，才是产品史真正接手的部分。
