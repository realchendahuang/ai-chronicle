过去，为一个模型准备任务，通常要在训练集和代码里动手。GPT-3 把这项工作的一部分搬进了输入框。

论文把评估分成三种情形。零样本只有任务说明；单样本再给一个示例；少样本则把若干输入与答案依次写进上下文。模型读完这些文字后接着生成，不进行梯度更新，参数也没有变化。这里的“学习”发生在一次推理的上下文里：示例临时规定了答案的格式、标签的意义，甚至规定了一种从未专门训练过的文字操作。

这使提示不再只是问题本身，也像一份小型数据集。LAMBADA 原本要求预测段落最后一个词，换一种填空格式并给出几个例子后，GPT-3 的少样本准确率达到 86.4%。在 TriviaQA 上，成绩从零样本的 64.3% 上升到少样本的 71.2%；在 WebQuestions 上，差距更大，从 14.4% 到 41.5%。同一篇论文中，少样本示例也被用来规定翻译方向、纠正文法、解释新造的词和完成简单代码。

1750 亿参数让这种现象变得醒目，却没有让表现整齐。GPT-3 在某些两位数算术上可靠，位数增加后迅速出错；在 WiC 词义判断上接近随机；常识与对抗性自然语言推理仍明显落后于专门系统。示例的写法和顺序会影响结果，模型究竟是在上下文中形成了怎样的内部任务，论文称之为尚未解释的问题。

训练语料的巨大范围又带来另一层不确定。测试题可能以近似形式出现在网络数据中，作者为数据污染做了专门分析，却无法为所有基准给出绝对排除。更广泛的偏差章节则留下了一句准确而沉重的概括：互联网训练的模型具有互联网尺度的偏差。规模放大了可调用的模式，也把其中的刻板印象、错误关联与有害表达一同带来。

OpenAI 没有发布 GPT-3 权重，而是通过 API 让外部开发者调用。于是，提示既成为操作模型的方法，也成为围绕同一基础模型开发产品的接口。人们无需拥有训练集群，甚至无需训练一个下游模型，就能用几行例子试出翻译器、分类器或写作工具。

GPT-3 最耐人寻味的地方不在于它给出多少正确答案，而在于任务定义的位置发生了移动。它从训练管线移到一段随时可以改写的文本里。这个文本很便宜、很灵活，也很脆弱；它让模型看似立刻领会要求，也让错误随着一个换序、一个改写或一道稍难的算术题显现出来。
