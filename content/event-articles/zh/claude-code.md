搜索代码。打开文件。修改。运行测试。读错误，再搜索一次。必要时提交，或者推送到 GitHub。

这些动作原本散落在编辑器、终端与浏览器里。2025 年 2 月 24 日，Anthropic 随 Claude 3.7 Sonnet 放出的 Claude Code 研究预览，把它们接成同一个命令行会话。官方发布页对它的描述很具体：能搜索和阅读代码，编辑文件，编写并运行测试，提交与推送代码，同时让开发者在每一步保持知情。它不再等待人把代码片段贴进聊天框，而是获得了对真实仓库采取动作的接口。开放形态是 limited research preview，不是“已可无人值守接管任意仓库”的成熟产品。

命令行作为界面，意味着 Claude Code 默认面向已会用 git 与测试命令的开发者，而不是纯图形化新手。权限提示把每次潜在破坏性操作变成可拒绝的事务；日志与 diff 把 Agent 叙事拉回可审查工件。与云端并行 Agent 相比，本地终端保留更低延迟的同步感，也承担本机环境被误操作的风险。研究预览标签要求读者降低“即刻替代团队”的预期，把事件理解为操作序列的产品化：读、改、跑、看，第一次被正式收成一条官方 CLI 故事。

终端的价值与黑色窗口更像不像工程师无关。软件会在这里反驳模型。文件不存在，命令立即失败；类型不对，编译器留下位置；测试没过，退出码不会因为解释流畅而改变。语言模型过去可以给出一段很像答案的补丁，Claude Code 则必须把补丁放进项目已有的约束里运行。一次执行产生新的状态，新状态又成为下一次判断的上下文。反馈回路扩大了模型能承担的工作，也把风险带得更近。

读取仓库与修改仓库不是同一权限，运行测试与执行任意命令也不是同一风险。Claude Code 的操作需要经过许可，高风险动作前请求确认；项目指令可以写入 `CLAUDE.md`，工具还可通过 GitHub 与 MCP 接入。所谓 Agent 能力从来不只属于模型，它由可见的上下文、授予的权限、可调用的工具和能够判定对错的测试共同组成。把“终端原生”读成“无需确认的完全自主”，会同时弄错产品机制与安全边界。

Anthropic 在首发时仍把它称为研究预览，并列出尚待改善之处：工具调用可靠性、长时间运行命令与应用内呈现。发布页给出的单次完成与节省时间案例来自内部或早期测试；SWE-bench 分数也依赖特定脚手架、筛选和重试口径。它们说明产品想往哪里去，不等于每一个陌生仓库都已经能被稳定接管。厂商自报的效率故事与基准，必须连同设置一起阅读。

Claude Code 使 AI 编程多了一种新的语法。开发者不再只问“这几行该怎么写”，也可以交代“查清失败原因，改完后跑测试”。但一句自然语言任务最终仍被拆回那些朴素动作：读、改、跑、看。模型的工作沿着终端展开，人的判断则留在授权与验收之处。“我已经完成”只是句子，文件差异、命令记录与测试结果才是两者之间能够传递的东西。研究预览的诚实之处，与编辑器内联补全不同，Claude Code 的会话以仓库为工作区，动作序列可被审计。`CLAUDE.md` 把项目约定从聊天记忆里挪到版本库可追踪的文件；权限提示则把“Agent”从修辞拉回操作系统现实。研究预览与 Claude 3.7 Sonnet 同日出现，说明模型能力与操作入口被绑成一套发布，而不是先有聊天再外挂终端。SWE-bench 等分数若出现在材料中，仍是特定脚手架下的厂商或合作评测，不能直接换算成你的单体仓库成功率。

把高风险操作做成确认对话框，是把安全左移到交互，而不是只靠模型自说“我很小心”。MCP 接入则让终端 Agent 可以调用外部工具生态，从而与协议层事件咬合。研究预览的边界包括工具可靠性与长命令体验，这些边界比宣传案例更值得写进正文：它们说明产品仍在学习如何与真实构建系统共处。

研究预览的诚实之处，是它仍然要求人站在那条传递链的终点。
