许多工程师仍把终端、Git 与测试命令当作真实开发的中心。Copilot 与 Cursor 把模型放进编辑器；Devin 把任务放进独立工作区。提示符闪烁的地方，才是状态、补丁与失败日志真正汇合之处。2025 年 2 月 24 日，Anthropic 随 Claude 3.7 Sonnet 放出 Claude Code 研究预览——一个在本地进程里读仓库、改文件、跑命令的命令行工具，并在较高风险操作前请求权限。

能力取决于权限、上下文、测试与人的监督，而不是“天然自主”。`CLAUDE.md` 可保存项目级指令；GitHub 与 MCP 工具可连接外部资源。产品单位从“建议一段代码”扩大到“可审查的操作序列”：查看状态、修改、执行、读错误、再重试。人仍在回路里，只是回路的默认界面换成了终端。每一次高风险操作前的停顿，是产品哲学，不是装饰。

研究预览标签要求把稳定性与覆盖范围读小。它演示的是一种协作节奏：模型接触完整反馈回路，人保留否决与授权。与云端并行任务队列相比，终端强调同步上下文；与纯补全相比，它强调执行后果。

Claude Code 没有取消代码审查。它把审查对象从补丁文本，部分扩展到命令与权限请求本身。习惯在 shell 里工作的人，会在熟悉的提示符旁，第一次认真考虑把 Agent 放进同一条路径——并记住：路径上的每一步，仍应能被拒绝。

终端 Agent 的风险模型与云端不同：它更接近开发者本机权限，误操作的后果也更直接。因此权限提示不是礼貌，而是产品边界。研究预览阶段，覆盖的仓库规模、语言与失败恢复策略仍在观察。它证明的不是“工程师可被替换”，而是反馈回路可以被模型触达——在人仍握有否决权的前提下。

本地进程意味着日志、密钥与未提交改动都在同一信任域。产品若只强调“能改仓库”，而不强调权限粒度，演示会好看，事故会难看。研究预览把能力与克制绑在一起发布，是这一事件值得记下的产品态度。

与编辑器插件相比，终端 Agent 更接近脚本与 Makefile 的世界：可组合、可管道、可进 CI 想象。研究预览还不是 CI 就绪声明，但方向已经指向“模型操作可脚本化”。脚本化之后，权限模型必须同样脚本化，否则自动化会放大误操作。
到 2025–2026 年，Claude Code 与编辑器 Agent、云端 Devin 类产品形成三角：终端强调本机反馈与权限停顿；编辑器强调索引与多文件；云端沙箱强调任务外包。三角不是互相取消，而是把“Agent Coding”拆成不同信任域。选型时真正要问的，是密钥、未提交 diff 与 CI 想像落在哪一侧。

把 Claude Code、Cursor、Devin 放进同一专题，不是为了找总冠军，而是为了看见产品单位如何迁移：补全 → 编辑器主交互 → 终端操作序列 → 沙箱任务。每一跳都留下未解决的验收问题。专题的密度来自对照，而不是单篇口号。
