NVIDIA 的 CUDA 工具包归档从 **1.0，2007 年 6 月**开始。可 CUDA 的历史通常从 **2006 年 11 月**算起，因为 NVIDIA 在那时公布了这套平台。两种日期并不冲突：前者是今天仍能在官方版本表上找到的首个工具包节点，后者是平台进入公众视野的起点。基础设施很少在某一天完整降临，它往往先被宣布，随后才以编译器、运行时、文档、驱动和库的形式逐项变得可用。官方时间线把它与 GPU 从图形加速器走向通用计算设备的过程写在同一条公司史里，而不是写成专为后来深度学习准备的预言。

在 CUDA 之前，GPU 已经擅长同时处理大量相似计算。图形渲染天然需要让许多像素和顶点并行通过相近的步骤，硬件因此积累了密集的算术单元。研究者也尝试用 GPU 做通用计算（GPGPU），但常要把问题伪装成图形任务，借着纹理、着色器和 OpenGL / DirectX 一类图形 API 表达原本与画面无关的算法。可计算，不等于可编程。错误难查，性能模型不透明，代码也难在团队间继承。

CUDA 改变的是这个接口。开发者用接近 C 的方式编写在 GPU 上运行的 kernel，把大量工作分配给线程；编程模型把线程块（block）、网格（grid）、共享内存和设备内存变成明确对象。编译器负责把代码送往 GPU，运行时处理设备与主机之间的数据搬运与协作，数学库逐渐封装常见线性代数与信号操作。并行硬件不再只通过图形管线露出一角，而成为一套能够学习、调试和版本化的软件平台。NVIDIA 公司时间线把它记为把 GPU 变成通用计算设备的关键节点之一。

CUDA 最初面向的远不止神经网络。科学计算、信号处理、分子模拟、计算金融和图像处理都能利用同一种大规模并行。神经网络后来与它相遇，是因为训练反复执行矩阵乘法与向量运算，工作负载恰好能铺到大量计算单元上。AlexNet 等 2012 年前后的工作已经示范 GPU 训练深度网络的收益；之后框架层把 CUDA 当作默认加速后端之一。2014 年前后推出的 cuDNN 等库又把卷积和其他神经网络原语进一步封装，框架作者不必为每代硬件重新从底层开始写 kernel。深度学习框架与 CUDA 工具链共同演化：一边是模型与自动微分，一边是驱动、编译器和厂商数学库。

稳定接口把速度变成了研究习惯。代码、教程、性能分析工具、库、人才与硬件采购相互加强：更多开发者带来更多软件，更多软件使下一块 NVIDIA GPU 更容易被采用。这种积累提高了效率，也建立了转换成本。后来所谓 NVIDIA 的护城河，包括多年兼容性、工具和开发者经验沉积出来的路径依赖，而不只有晶体管。竞争者可以做出加速器；更难复制的是一整条可继承的软件地层。

深度学习的突破经常以论文与模型命名，CUDA 则位于较低的一层。公式给出扩展的可能，并行线程、内存层次、驱动版本和数学库把这种可能整理成日常工程。CUDA 的线程层次也改变了性能调优的日常语言：占用率、共享内存银行冲突、主机—设备拷贝开销、流（stream）与异步执行，成为论文附录与工程 checklist 里的常见项。早期 GPGPU 用图形 API 时，这些概念往往被藏在着色器技巧里；平台化之后，它们变成可教学、可文档化的对象。cuBLAS、cuFFT 等库先服务科学计算，cuDNN 再把卷积等原语接进深度学习框架，显示生态是分层长出来的，而不是为某一篇神经网络论文一次性定制。

版本归档里那串从 1.0 开始的数字看似只是旧软件下载清单，逐项连起来，却记录了算力如何变成可继承的软件环境——以及这条环境如何在没有宣称“智能”的情况下，成为后来智能系统扩张时几乎绕不开的物质条件。芯片可以迭代，接口若断裂，研究代码的继承链就会断。CUDA 赌的是接口尽量不断。
