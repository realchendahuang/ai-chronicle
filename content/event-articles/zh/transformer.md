《Attention Is All You Need》像一句故意说满的话。论文做的事却首先是删减：在当时最成熟的机器翻译架构里，编码器和解码器通常依靠 RNN 或 LSTM 逐词推进，注意力只是架在循环网络之上的辅助模块。Transformer 留下注意力，拿掉了循环，也没有用卷积接替它。

循环网络的时间步与词序绑在一起。第十个 token 的隐藏状态要等前九个依次算完，两个相距很远的位置交换信息，也必须经过一长串中间状态。自注意力把这件事改成一次全局读取：序列中每个位置产生 query、key 和 value，用 query 与所有 key 的匹配程度加权汇总 value。任意两个位置之间的计算路径缩短到常数级，各个位置又可以同时处理，更适合 GPU 的矩阵运算。

一次注意力容易把不同关系平均在一起，于是论文把表示投影到多个子空间，让多个 head 并行关注不同位置和关系，再把结果拼回。模型没有循环，也就不再天然知道先后顺序；正弦和余弦构成的位置编码被加到词嵌入上，把位置信息补回计算。六层编码器、六层解码器中，每个注意力或前馈子层外还有残差连接和层归一化。所谓“只要注意力”，并不意味着架构里真的只剩一个部件。

论文用机器翻译而非开放式生成证明这套设计。大模型在 WMT 2014 英德翻译上得到 28.4 BLEU，比此前最佳结果高出两个以上 BLEU；英法翻译达到 41.8。训练使用八块 P100 GPU，耗时三天半。比成绩更关键的是计算形状变了：模型不再必须沿句子长度串行等待，扩大数据和参数时，并行硬件终于能被更充分地使用。

这项交换有明确价格。完整自注意力要比较所有位置对，序列长度为 n 时，时间和内存中的关键项按 O(n²) 增长。短句翻译里它很划算，长文档则会迅速昂贵。

后来仅保留编码器或仅保留解码器的模型，把 Transformer 推到语言之外；而稀疏注意力、线性注意力和长上下文工程，又不断回到论文那张复杂度表，处理当初为了去掉串行计算而接受的这一笔账。
