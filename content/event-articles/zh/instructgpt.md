InstructGPT 训练流程的人工环节，由一支大约四十人的标注团队完成。

他们先写答案。面对来自 OpenAI API 和标注员自拟的指令，标注员示范一个理想回应，研究者用这些示范对 GPT-3 做监督微调。论文列出的监督训练数据包括 11,295 条标注员提示和 1,430 条客户提示。模型由此先学会一种基本姿态：输入不一定是等待续写的文章，也可能是需要执行的要求。

接着，人类不再写唯一答案，而是比较。系统为同一提示生成四到九个候选，标注员把它们排序。研究者用这些比较训练一个奖励模型，让它估计人更偏好哪种输出。最后，PPO 以奖励模型为评分者继续优化语言模型，同时加入预训练目标，减轻某些公共 NLP 任务上的性能退化。

这条生产线改变了“更好”的含义。预训练优化下一个 token 是否更可能出现；InstructGPT 优化的是一组人面对完整回答时的选择。结果很醒目：在论文的 API 提示测试集上，1.3B 参数的 InstructGPT 输出比原始 175B GPT-3 更受标注员偏好。175B InstructGPT 与同尺寸 GPT-3 相比，胜率为 85±3%；即使给后者少样本提示，前者仍有 71±4% 的偏好率。在封闭域任务里，编造输入之外信息的比例从 GPT-3 的 41% 降到 21%。

但偏好不是事实判定器。InstructGPT 仍会捏造事实、误解带有错误前提的问题，也会用很长的保留语回答简单问题。毒性有所下降，论文没有观察到偏差指标的显著改善。为了讨好奖励模型而牺牲原有任务能力的现象，被作者称为“alignment tax”；混合预训练梯度可以缓解，却不能把所有回退抹掉。

更重要的限定写在讨论部分。研究者说，他们对齐的是特定标注员与研究人员影响下的偏好，不是笼统的“人类价值”。标注规范由 OpenAI 制定，训练数据以英语为绝大多数，标注员之间的一致率约为 73%。换一批人、换一种文化背景、换一套规则，“好回答”的边界可能移动。

InstructGPT 把一个新问题带到模型核心：能力之外，回答方式也可以被训练。直接、诚实、谨慎、拒绝哪些要求，都不再只是提示技巧，而成为数据与优化目标的一部分。聊天产品后来会把这种方式扩展到数以百万计的交互；在论文的剖面里，仍能看见最初那条人工流水线，以及每一个“更符合人意”背后不可省略的问题——符合谁的意。
