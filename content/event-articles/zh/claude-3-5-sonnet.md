模型家族的命名曾经很像一栋层次清楚的楼：最大的模型住在顶层，最贵、最慢，也被默认最聪明；小模型在下面，用能力换速度和价格。Claude 3.5 Sonnet 让这张楼层图开始失真。

Anthropic 发布它时，定价仍属于 Sonnet 档——每百万输入 token 3 美元、输出 token 15 美元，支持 20 万 token 上下文——却把比较对象直接指向上一代旗舰 Claude 3 Opus。公司称它的运行速度约为 Opus 的两倍，并公布了推理、知识、视觉和代码评测上的提升。在一项内部 Agent 编程评估中，官方报告 Sonnet 解决 64% 的任务，Claude 3 Opus 为 38%。这些数字说明 Anthropic 怎样评价自己的新模型；测试集、工具配置与评分方式决定了它们的边界，不能替代所有真实工作负载的比较。

排行榜只解释了这次发布的一部分。大模型进入生产以后，所谓能力从来不是单个最高分。一个请求要等待多久，一天可以调用多少次，长文档是否放得进去，代码修改能否稳定执行，都会参与最后的选择。最强模型如果昂贵到只能偶尔请教，便很难成为工作流的底座。Claude 3.5 Sonnet 把 Anthropic 所称的旗舰级表现放进更实用的延迟和成本，使“主力模型”成为比“最大模型”更有意义的称呼。

同一天出现的 Artifacts 给这种变化补上了产品形状。模型生成的代码、文档或网页不再只沉入对话记录，而是在旁边的独立区域中显示，可以继续查看和编辑。聊天的产物第一次被明确当作工作对象，而不只是消息。这个界面并没有自动赋予模型更强推理能力，却改变了用户与结果的关系：一边谈，一边看见东西成形；不满意时改的是作品，不必先在长对话里寻找上一版。

Claude 3.5 Sonnet 因此位于两个转变的交点。一个发生在模型供应上：厂商开始反复改善能力、速度和价格的整条曲线，而不是只在最昂贵的一端争夺第一。另一个发生在界面里：回答逐渐变成可以操作、保存、迭代的对象。

它没有让模型等级从此消失。旗舰仍会向更难的任务伸展，小模型也继续争夺效率。只是从这一代 Sonnet 起，产品团队越来越少问“哪一个型号最大”，而开始问一个更像日常生活的问题：哪一个愿意一直开着？
