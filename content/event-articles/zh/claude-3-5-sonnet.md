模型家族的命名曾经很像一栋层次清楚的楼：最大的模型住在顶层，最贵、最慢，也被默认最聪明；小模型在下面，用能力换速度和价格。Claude 3.5 Sonnet 让这张楼层图开始失真。

2024 年 6 月 20 日，Anthropic 发布它时，API 定价仍属于 Sonnet 档——每百万输入 token 3 美元、输出 token 15 美元，与 Claude 3 Sonnet 同级，并支持 20 万 token 上下文——却把比较对象直接指向上一代旗舰 Claude 3 Opus。公司称它的运行速度约为 Opus 的两倍，并公布了推理、知识、视觉和代码评测上的提升。在一项内部 Agent 编程评估中，官方报告 Sonnet 解决 64% 的任务，Claude 3 Opus 为 38%。这些数字说明 Anthropic 怎样评价自己的新模型；测试集、工具配置与评分方式决定了它们的边界，不能替代所有真实工作负载的比较，也不是独立第三方复现的结论。

将 Sonnet 与上一代 Opus 对打，是定价心理学与技术叙事的双重操作：顾客听到“更强”，账单仍按中档走。对创业公司与高频 Agent 循环，这种组合比偶尔调用顶级旗舰更决定能否上线。视觉能力与代码能力被放在同一发布页，说明 2024 年中的“主力模型”已被期望同时处理截图与仓库。Artifacts 让 Claude.ai 的会话产出物可被当作工件迭代，缩小了聊天工具与轻量创作工具的距离。中档之名、主力之实，是这一天最清晰的产品句子。

排行榜只解释了这次发布的一部分。大模型进入生产以后，所谓能力从来不是单个最高分。一个请求要等待多久，一天可以调用多少次，长文档是否放得进去，代码修改能否稳定执行，都会参与最后的选择。最强模型如果昂贵到只能偶尔请教，便很难成为工作流的底座。Claude 3.5 Sonnet 把 Anthropic 所称的旗舰级表现放进更实用的延迟和成本，使“主力模型”成为比“最大模型”更有意义的称呼。它在 Claude.ai 与 API 上线，免费用户亦可在产品限制下使用——开放形态是“可用的中坚”，而不是“仅企业旗舰试用”。

同一天出现的 Artifacts 给这种变化补上了产品形状。模型生成的代码、文档或网页不再只沉入对话记录，而是在旁边的独立区域中显示，可以继续查看和编辑。聊天的产物第一次被明确当作工作对象，而不只是消息。这个界面并没有自动赋予模型更强推理能力，却改变了用户与结果的关系：一边谈，一边看见东西成形；不满意时改的是作品，不必先在长对话里寻找上一版。Artifacts 以预览能力进入 Claude.ai，与模型能力发布绑在同一天，却属于不同的产品层。

Claude 3.5 Sonnet 因此位于两个转变的交点。一个发生在模型供应上：厂商开始反复改善能力、速度和价格的整条曲线，而不是只在最昂贵的一端争夺第一。另一个发生在界面里：回答逐渐变成可以操作、保存、迭代的对象。

它没有让模型等级从此消失。旗舰仍会向更难的任务伸展，小模型也继续争夺效率。只是从这一代 Sonnet 起，产品团队越来越少问“哪一个型号最大”，而开始问一个更像日常生活的问题：哪一个愿意一直开着？答案若只看参数量或家族序号，会答错；若看延迟、单价、上下文与一次工作能否做完，价格表上的 3/15 美元并不只是财务细节：它允许团队把高质量模型放进高频自动路径，而不是只在人工审批的关键步骤调用旗舰。200K 上下文使长仓库、长文档与多文件讨论更容易单次塞进窗口，尽管“放得进”仍不等于“都读得准”。Artifacts 预览则把 Claude.ai 从纯对话推向可迭代工作台，与 API 侧的模型升级形成消费级与开发级的双线叙事。厂商基准中的 64% 对 38% 必须连同内部 Agent 编程评估口径阅读——它是定位声明，不是跨公司排行榜。

免费档与付费档共享模型家族时，速率与功能差异仍在，但“手边模型够强”的感受会跨过付费墙传播。这对竞争对手构成价格—能力曲线压力：旗舰溢价必须证明自己在延迟与成本之外仍不可替代。3.5 Sonnet 之后，中档主力化成为行业默认策略之一，而不只是 Anthropic 的一次促销。

若看延迟、单价、上下文与一次工作能否做完，会更接近 2024 年夏天人们实际做的选择。
