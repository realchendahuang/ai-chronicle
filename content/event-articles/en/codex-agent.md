A coding agent in a local terminal occupies a continuous stretch of time. A person assigns work, watches search and edits, and intervenes at forks. On 16 May 2025, OpenAI's cloud Codex research preview in ChatGPT rearranged that time: tasks leave, each enters an isolated environment, several can run at once, and patches return later with evidence.

After connecting a GitHub repository, Codex prepares a container preloaded with code for each task, reads files, runs commands, edits code, and executes tests. The first version was driven by codex-1. One agent cannot see what another is changing; each job owns its environment and process. Isolation makes parallelism possible and writes task boundaries more clearly. Fix a bug, add tests, explain a passage of code—three independent commissions rather than a single conversation queue. Access was a research preview for ChatGPT users with the relevant entitlements, not a universal default “cloud engineer” for every developer.

Cloud isolation lowers direct laptop risk and raises governance questions about connected repos and injected secrets. Parallel tasks reward clean work-order cuts and punish bad splits with merge and semantic conflicts. Research-preview honesty emphasized patches and logs rather than “engineers replaced.” Reusing the 2021 name warns readers to check the date: an API model and a cloud task system are a product philosophy apart.

The cloud did not turn software engineering into one sentence. It moved many intermediate moments off the human desktop. Agents can still misread requirements; tests can still under-cover; unwritten repository conventions can still be ignored. OpenAI therefore put logs, test results, and code citations into the delivery, so users can reconstruct what happened rather than receive only a completion claim. Tasks run in the background; responsibility is not hosted with them. Parallelism count is not capability: ten simultaneous failures only manufacture ten review-worthy noise packets faster.

The relationship to terminal agents is complementary, not exclusive. Terminals suit correction-in-flight when goals are still moving; cloud tasks suit work that is well bounded, isolatable, and finally checkable. One is a shared workbench; the other is a stack of work orders. Model capability may be similar; collaboration institutions differ—synchronous context versus asynchronous evidence. This product also must not be confused with the 2021 Codex API model that shared the name: that generation turned natural language into callable code; this one turns software tasks into parallel cloud jobs that return patch evidence.

Codex brought queue management into everyday development: which tasks are independent enough, which can start together, what counts as pass, how conflicts re-enter the main branch. When multiple agents work, time to generate code may no longer be the scarcest resource. Cutting work correctly, writing acceptance clearly, and knowing which of a batch of seemingly finished patches deserves merge take more weight. The first research preview did not deliver “parallel engineers.” It delivered a more restrained product unit: a software task with its own environment, process record, and return package.
